A comprehensive step-by-step guide to get your static web application live on Azure

What is Azure Static Web Apps?

Azure Static Web Apps is a service that automatically builds and deploys full stack web apps to Azure from a code repository. It's perfect for static sites, SPAs, and JAMstack applications.

Deployment Steps

1

Prerequisites

  • Azure account (free tier available)
  • GitHub account
  • Your web application code ready
2

Prepare Your Repository

  • Push your code to a GitHub repository
  • Ensure your main files (index.html, etc.) are in the root or a specific folder
  • Make sure your repository is public or you have appropriate permissions
3

Create Static Web App in Azure

  • Go to the Azure Portal
  • Click "Create a resource"
  • Search for "Static Web Apps" and select it
  • Click "Create"
4

Configure Your Static Web App

  • Subscription: Select your Azure subscription
  • Resource Group: Create new or select existing
  • Name: Give your app a unique name
  • Plan type: Choose Free for development
  • Region: Select the closest region to your users
5

Connect to GitHub

  • Click "Sign in with GitHub" and authorize Azure
  • Organization: Select your GitHub username/organization
  • Repository: Choose your repository
  • Branch: Select the branch to deploy (usually main/master)
6

Set Build Details

  • Build Presets: Select your framework (e.g., Custom, React, Angular, Vue)
  • App location: Path to your app code (usually "/" or "/src")
  • API location: Leave empty for static sites
  • Output location: Build output folder (e.g., "dist", "build", or "/" for static HTML)
7

Review and Create

  • Review all your settings
  • Click "Review + create"
  • Click "Create" to deploy your Static Web App
8

Automatic Deployment

  • Azure automatically creates a GitHub Action workflow
  • The deployment process starts automatically
  • Monitor progress in the GitHub Actions tab
  • Deployment typically takes 2-5 minutes
9

Access Your Live Site

  • Go to your Static Web App resource in Azure Portal
  • Copy the URL from the "Overview" section
  • Your site is now live and accessible worldwide!

ðŸ”§ Troubleshooting

Build Failed?

Check the GitHub Actions logs for detailed error messages. Ensure your build commands and output location are correct.

404 Errors?

Verify your app location and output location settings. For SPAs, you may need to configure fallback routes.

Deployment Stuck?

Check if your repository permissions are correct and the GitHub integration is properly authorized.
